## 3.1 系统需求分析

**3.1.1 功能需求分析（约 1 页）**
- 用用例图展示系统主要功能
- 两个角色：医生（客户端）、管理员（服务端）
- 医生侧：患者管理、智能分诊、风险评估、多专家会诊、知识查询、对话记录
- 管理员侧：PDF 文献上传、知识图谱构建、图谱可视化、文档管理、数据导出
- 配一张功能需求列表表格（编号/名称/描述/优先级）

**3.1.2 非功能需求分析（约 0.5 页）**
- 性能：单次分诊 <5 秒、知识图谱查询 <2 秒、支持 10+ 并发
- 可用性：界面简洁直观、操作流程清晰
- 安全性：患者数据隔离、API 访问控制
- 可扩展性：模块化设计、支持新增疾病领域

## 3.2 系统总体架构设计（约 2 页）

- 描述三层架构 + 模型服务层：
  - **客户端层**：多智能体协作引擎 + Vue3 诊断界面
  - **服务端层**：REST API + MCP 工具服务 + 知识图谱构建 + Vue3 管理界面
  - **数据层**：Neo4j（知识图谱）+ Redis（向量索引）+ 文件系统
  - **模型服务层**：AutoDL + Ollama + cpolar 穿透
- 通信方式：REST API、MCP(SSE)、SSE 流式推送
- 配系统总体架构图（最重要的图之一）

## 3.3 功能模块设计（约 5 页）

**3.3.1 知识图谱构建模块**
- 七步流水线：PDF 解析→HTML 清洗→Markdown 转换→文本分块→实体关系抽取→Neo4j 导入→向量化
- 每步的输入/输出
- 重点解释实体关系抽取的 Prompt 设计

**3.3.2 多智能体工作流模块**
- 五节点职责划分
- supervisor 路由逻辑：根据输入类型分发任务
- 上下文感知路由：has_triaged / has_diagnosis 状态判断

**3.3.3 MCP 工具服务模块**
- 4 个 MCP 工具定义及输入参数/返回格式
- SSE 传输协议

**3.3.4 服务端知识管理模块**
- 10+ 个 REST API 接口
- 文档管理：上传/列表/加载/删除/导出
- 异步构建 + SSE 实时进度推送
- 数据一致性：Neo4j + Redis + 文件系统三方同步

**3.3.5 分诊评估模块**
- 并行双智能体：医学顾问 + 急诊分诊
- 五级分诊标准
- 综合分析节点融合策略

**3.3.6 风险评估模块**
- 症状-疾病匹配 → Softmax 概率计算 → 诊断检查推荐

**3.3.7 多专家会诊模块**
- 三专家：诊断/治疗/影像
- RAG 知识检索 + 综合报告

**3.3.8 知识查询模块**
- ReAct 智能体 + MCP 工具集成调用

**3.3.9 患者数据管理模块**
- CRUD 操作 + 问诊记录关联存储
