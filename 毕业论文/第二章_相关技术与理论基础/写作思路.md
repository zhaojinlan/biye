## 2.2 大语言模型与提示工程（约 2 页）

**第一部分：什么是大语言模型**
- 通俗解释：LLM 是基于深度学习的自然语言处理模型，通过海量文本训练，具备理解和生成自然语言的能力
- LLM 发展脉络一句话概括：从 BERT 到 GPT 系列，再到国产开源模型 Qwen
- 本系统使用 Qwen2.5-14B：阿里通义千问开源模型，支持中英双语
- 部署方式：AutoDL 云 GPU + Ollama 部署 + cpolar 内网穿透暴露为公网 API（OpenAI 兼容接口）

**第二部分：什么是提示工程**
- 通过设计输入文本（提示词）来引导 LLM 输出期望结果
- 本系统用到的提示技术（挑重点）：
  - **系统提示词**：给每个智能体设定不同的医学角色（如"你是一名急诊分诊护士"）
  - **结构化输出**：要求 LLM 返回 JSON 格式结果（如实体抽取）

## 2.3 多智能体系统与 LangGraph 框架（约 2 页）

**第一部分：什么是智能体和多智能体系统**
- 智能体（Agent）：能感知环境、做出决策并执行行动的自主程序
- 多智能体系统（MAS）：多个智能体协作完成复杂任务
- 本系统采用"监督者-工作者"协作模式：一个监督节点负责分配任务，4 个专业节点各司其职
- 类比：就像医院里的导诊台（监督节点）把患者分配到不同科室（工作节点）

**第二部分：LangGraph 框架简介**
- LangGraph 是基于有向图的多智能体编排框架
- 核心概念通俗解释：节点 = 一个智能体，边 = 流转关系，状态 = 共享数据
- 为什么用 LangGraph：医疗诊断有明确流程步骤，用图编排最直观
- 支持多轮对话记忆（Checkpointer），适合问诊场景

## 2.4 知识图谱与 Neo4j 图数据库（约 2 页）

**第一部分：什么是知识图谱**
- 用图的方式组织知识：实体作为节点，关系作为边
- 举例：(感冒)--[症状]-->(发热)
- 医学知识图谱的意义：将散落在文献中的知识结构化

**第二部分：知识图谱如何构建**
- 一般流程：文本获取 → 实体识别 → 关系抽取 → 存入数据库
- 本系统特点：使用 LLM 自动完成实体识别和关系抽取（零样本）

**第三部分：Neo4j 图数据库**
- 最流行的图数据库，专门存储和查询图结构数据
- Cypher 查询语言举例：`MATCH (d:Disease)-[:HAS_SYMPTOM]->(s:Symptom) RETURN d, s`
- 本系统存储 6 种节点（Disease/Symptom/RiskFactor/Pathogen/Treatment/DiagnosticTest）

## 2.5 检索增强生成 RAG（约 1.5 页）

**第一部分：为什么需要 RAG**
- LLM 的局限：知识截止日期、可能产生"幻觉"、无法获取私有数据
- RAG 核心思想：先检索相关知识，再让 LLM 参考这些知识来回答
- 类比：考试时允许翻书

**第二部分：RAG 怎么工作**
- 基本流程：提问 → 向量化 → 检索 → 增强 Prompt → LLM 生成
- 文本向量化：M3E-Base 把文字转成 768 维向量
- 向量检索：余弦相似度找最相关内容

**第三部分：本系统的混合检索**
- 同时使用两种检索：Neo4j 知识图谱精确查询 + Redis 向量检索语义匹配
- 不同场景不同策略：风险评估用图谱，会诊用向量检索

## 2.6 模型上下文协议 MCP（约 1 页）

- MCP 是什么：Anthropic 发布的开放协议，让 AI 智能体以统一标准接口调用外部工具
- 类比：USB 接口让各种设备用同一种方式连接电脑
- 三个角色：Host（智能体）→ Client（通信层）→ Server（工具服务）
- 本系统应用：服务端通过 MCP 暴露 4 个医学知识查询工具，客户端智能体自动发现并调用
