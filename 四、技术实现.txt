基于多智能体与知识图谱的智能医疗辅助诊断系统

四、技术实现

1. 总体架构

本系统采用微服务架构，分为客户端服务和服务端两大部分：

客户端服务（medical_client）：
- 功能：多智能体协作引擎、患者数据管理、对话流程控制
- 技术栈：Python、FastAPI、LangChain、LangGraph
- 部署：独立部署，可打包为可执行文件
- 通信：通过MCP协议调用服务端工具

服务端（medical_server）：
- 功能：知识图谱管理、文献处理、数据存储、MCP工具服务
- 技术栈：Python、FastAPI、Neo4j、Redis、Docling
- 部署：服务器部署，支持Docker容器化
- 通信：提供MCP工具接口和REST API

数据层：
- Neo4j图数据库：存储医学知识图谱（疾病、症状、风险因子、治疗方法等实体及关系）
- Redis数据库：存储向量索引、缓存数据
- 文件系统：存储原始PDF文献、处理后的文档、患者数据


2. AI关键技术应用

技术一：大语言模型（LLM）
- 模型：Qwen2.5-14B（通过OpenAI兼容接口调用）
- 应用场景：
  * 任务分类与路由：监督节点使用LLM判断用户输入类型，路由到合适的处理节点
  * 医学实体识别：从文献中抽取疾病、症状、药物等医学实体
  * 关系抽取：识别实体间的医学关系（如疾病-症状、疾病-治疗方法）
  * 诊断推理：结合检索到的知识进行医学推理和诊断建议
  * 报告生成：生成结构化的分诊报告、诊断报告和会诊报告
- 优化策略：
  * 针对不同节点设置差异化的温度参数（分诊节点temperature=0.1，确保稳定输出）
  * 使用提示工程技术，设计专业的医学提示词模板
  * 限制最大输出长度，避免生成冗余内容

技术二：多智能体系统（Multi-Agent System）
- 框架：LangGraph
- 架构设计：
  * 监督节点（Supervisor Node）：任务分类与路由中心
  * 分诊节点（Triage Node）：并行分诊评估
  * 风险评估节点（Recommend Node）：疾病风险分析与检查建议
  * 多专家会诊节点（Experts Node）：多视角诊断分析
  * 知识查询节点（Query Node）：医学知识问答
- 协作机制：
  * 使用状态图（StateGraph）管理智能体间的数据流转
  * 通过共享状态（State）实现上下文传递
  * 支持条件路由，根据任务类型动态选择执行路径
- 优势：
  * 专业化分工，每个智能体专注特定任务
  * 并行处理，提高系统响应速度
  * 可扩展性强，易于添加新的专业智能体

技术三：知识图谱（Knowledge Graph）
- 数据库：Neo4j
- 图谱结构：
  * 节点类型：疾病（Disease）、症状（Symptom）、风险因子（RiskFactor）、病原体（Pathogen）、治疗方法（Treatment）、检查方法（DiagnosticTest）
  * 关系类型：HAS_SYMPTOM、HAS_RISK_FACTOR、HAS_PATHOGEN、HAS_TREATMENT、REQUIRES_TEST等
- 构建流程：
  * PDF文献解析：使用Docling将PDF转换为结构化HTML
  * 文本清洗：去除无关内容（封面、参考文献等），保留核心医学知识
  * 文本分块：使用RecursiveCharacterTextSplitter进行智能分块（chunk_size=2000）
  * 实体抽取：使用LLM从每个文本块中抽取医学实体和关系
  * 图谱存储：将抽取结果存入Neo4j，自动去重和合并
- 查询能力：
  * 基于症状查询相关疾病
  * 查询疾病的风险因子、病原体、治疗方法
  * 多跳推理（如：症状→疾病→治疗方法）
  * Cypher查询语言支持复杂的图遍历

技术四：检索增强生成（RAG）
- 向量模型：M3E-Base（中文医学文本嵌入模型）
- 向量数据库：Redis（使用RediSearch模块）
- 检索策略：
  * 症状向量检索：将患者症状描述向量化，检索相似症状
  * 文档向量检索：从医学文献中检索相关段落
  * 混合检索：结合知识图谱的精确查询和向量检索的语义匹配
- 应用场景：
  * 多专家会诊节点：检索相关医学文献支持专家意见
  * 知识查询节点：回答医学知识问题
  * 风险评估节点：检索疾病相关的诊断标准和指南
- 优化：
  * 文档分块策略优化（chunk_size=1500，chunk_overlap=150）
  * Top-K检索（默认返回3-5个最相关结果）
  * 结果重排序，优先返回高质量内容

技术五：模型上下文协议（MCP）
- 协议：Model Context Protocol
- 实现：FastMCP框架
- 工具定义：
  * symptom_search_analyze：症状检索与疾病分析
  * query_disease_info：查询疾病详细信息
  * get_diagnostic_tests：获取疾病的诊断检查方法
  * query_treatment_methods：查询治疗方法
  * rag_search：向量检索医学文档
- 通信方式：
  * 传输协议：Server-Sent Events（SSE）
  * 数据格式：JSON
  * 连接管理：长连接，支持流式响应
- 优势：
  * 标准化的工具调用接口
  * 支持工具发现和动态加载
  * 轻量级客户端，重量级服务端分离


3. AI工具应用说明

开发框架与库：
- LangChain（v0.3.27）：LLM应用开发框架，提供模型封装、提示管理、链式调用等功能
- LangGraph（v0.6.8）：多智能体工作流框架，基于状态图实现复杂的智能体协作
- FastMCP（v0.1.0）：MCP协议实现框架，简化工具服务开发

文档处理工具：
- Docling（v2.0.0）：IBM开源的文档智能解析工具，支持PDF、Word等格式
- BeautifulSoup4：HTML解析库，用于清洗和提取文档内容
- html2text：HTML到Markdown转换工具

机器学习工具：
- Sentence-Transformers（v2.2.0+）：文本向量化工具
- Transformers（v4.35.0+）：Hugging Face模型库
- PyTorch：深度学习框架（支持GPU加速）

数据库工具：
- Neo4j Python Driver：Neo4j官方Python客户端
- py2neo：Neo4j的高级Python接口
- Redis-py：Redis官方Python客户端
- RediSearch：Redis的全文检索和向量检索模块

Web框架：
- FastAPI（v0.117.1）：现代化的Python Web框架，支持异步和自动API文档
- Uvicorn：ASGI服务器，用于运行FastAPI应用
- SSE-Starlette：Server-Sent Events支持

其他工具：
- LlamaIndex：文档加载和索引工具
- ONNX Runtime：模型推理加速（用于OCR）
- RapidOCR：快速OCR识别工具


4. 核心模块说明

模块一：多智能体工作流引擎（Agent/flow.py）
- 功能：协调多个智能体完成复杂医疗任务
- 核心组件：
  * StateGraph：状态图管理器
  * 5个专业化节点：supervisor、triage、recommend、experts、query
  * 路由函数：根据任务类型动态选择执行路径
- 工作流程：
  1. 用户输入进入监督节点
  2. 监督节点分析输入类型，路由到对应节点
  3. 专业节点处理任务，更新状态
  4. 返回结果给用户，支持多轮对话

模块二：知识图谱构建流程（Construct/knowledge_workflow.py）
- 功能：自动化处理医学文献，构建知识图谱
- 处理流程：
  1. PDF解析：使用Docling转换为HTML
  2. 文本清洗：去除无关内容，保留核心知识
  3. 文本分块：智能分割长文档
  4. 实体抽取：使用LLM识别医学实体和关系
  5. 图谱存储：写入Neo4j数据库
  6. 向量索引：创建症状向量索引
- 输出：
  * Neo4j知识图谱
  * 处理日志和中间文件（HTML、Markdown、JSON）
  * 元数据文件

模块三：MCP工具服务（MCP/mcp_server.py）
- 功能：提供标准化的医学知识查询工具
- 工具列表：
  * symptom_search_analyze：症状检索与分析
  * query_disease_info：疾病信息查询
  * get_diagnostic_tests：诊断检查方法查询
  * query_treatment_methods：治疗方法查询
  * rag_search：文档向量检索
- 实现方式：
  * 使用FastMCP框架定义工具
  * 连接Neo4j和Redis数据库
  * 提供SSE接口供客户端调用

模块四：分诊节点（Agent/triage_node.py）
- 功能：并行分诊评估
- 设计：
  * 智能体1（医学顾问）：基于MCP工具进行症状分析
  * 智能体2（急诊分诊）：基于五级分诊标准评估
  * 综合分析：整合两个视角的结果
- 输出：
  * 分诊级别（Ⅰ-Ⅴ级）
  * 建议科室
  * 危重指征说明
  * 需要询问的补充信息

模块五：风险评估节点（Agent/recommend_node.py）
- 功能：疾病风险分析与检查建议
- 核心算法：
  * 症状-疾病匹配：基于知识图谱查询相关疾病
  * 风险因子评估：计算患者匹配的风险因子数量
  * 概率计算：使用Softmax算法量化疾病概率
  * 检查建议：基于疾病推荐诊断检查
- 输出：
  * 最可能的疾病及置信度
  * 所有候选疾病的概率分布
  * 推荐的诊断检查项目

模块六：多专家会诊节点（Agent/experts_node.py）
- 功能：模拟多学科专家会诊
- 专家角色：
  * 诊断专家：分析症状和检查结果，给出诊断意见
  * 治疗专家：基于诊断结果提供治疗方案
  * 影像专家：解读影像学检查结果
- 工作流程：
  1. RAG检索：为每个专家检索相关医学知识
  2. 并行分析：三个专家独立给出意见
  3. 综合报告：整合多专家意见，生成会诊报告
- 输出：
  * 各专家的独立意见
  * 综合诊断结论
  * 治疗建议
  * 进一步检查建议

模块七：知识查询节点（Agent/query_node.py）
- 功能：医学知识问答
- 实现：
  * 使用create_react_agent创建ReAct智能体
  * 集成MCP工具进行知识检索
  * 支持复杂的多步推理
- 应用场景：
  * 疾病知识查询
  * 诊疗指南查询
  * 医学术语解释
  * 评分标准说明

模块八：向量检索系统（RAG/tools/rag.py）
- 功能：基于Redis的文档向量检索
- 核心功能：
  * 文档加载与分块
  * 文本向量化（使用M3E模型）
  * 向量索引创建
  * 相似度检索
- 性能优化：
  * 使用ONNX Runtime加速推理
  * 批量向量化处理
  * Redis向量索引优化

模块九：知识图谱查询（RAG/tools/KGQuery.py）
- 功能：封装Neo4j知识图谱查询
- 查询类型：
  * 基于症状查询疾病
  * 查询疾病的风险因子
  * 查询疾病的病原体
  * 查询治疗方法
  * 查询诊断检查
- 特点：
  * 使用Cypher查询语言
  * 支持模糊匹配
  * 自动去重和聚合
  * 日志记录和错误处理
