## 图 3-6 风险评估算法流程

```mermaid
graph TD
    IN[患者病史信息] --> EXT[LLM提取<br/>症状关键词]
    EXT --> MATCH[Neo4j症状匹配<br/>向量相似度检索]
    MATCH --> SCORE[计算疾病匹配度<br/>Softmax概率分布]
    SCORE --> TOP[取Top-N<br/>最可能疾病]
    TOP --> DIAG[查询推荐检查<br/>MCP工具调用]
    DIAG --> OUT[输出风险评估报告<br/>疾病概率+推荐检查]
```

## 图 3-8 一次完整分诊请求的调用时序图

```mermaid
sequenceDiagram
    participant U as 用户(Vue3前端)
    participant C as 客户端FastAPI
    participant LG as LangGraph引擎
    participant LLM as Qwen2.5-14B<br/>(AutoDL)
    participant MCP as MCP Server
    participant DB as Neo4j/Redis

    U->>C: POST /api/chat/stream<br/>"患者出现胸痛、呼吸困难"
    C->>LG: 调用graph.astream()
    LG->>LLM: 监督节点：任务分类
    LLM-->>LG: 返回 "triage_node"
    LG->>LLM: 分诊节点(并行)：<br/>医学顾问 + 急诊分诊
    LLM-->>LG: 返回分诊结果
    LG->>MCP: 调用 symptom_search_analyze
    MCP->>DB: Cypher查询 + 向量检索
    DB-->>MCP: 返回匹配结果
    MCP-->>LG: 返回工具结果
    LG->>LLM: 综合分析生成报告
    LLM-->>LG: 返回最终报告
    LG-->>C: 流式返回结果
    C-->>U: SSE逐字推送AI回复
```
